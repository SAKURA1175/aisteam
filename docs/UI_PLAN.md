# TutorMarket AI UI 计划书

## 1. UI 目标

UI 的目标不是“好看”这么简单，而是：

- 让用户快速理解老师和知识的关系
- 让聊天体验有教学感，而不是普通客服感
- 让记忆、引用、规则、知识库这些复杂能力看起来清楚
- 让后台配置足够高效

## 2. UI 设计方向

整体风格建议：`教育科技 + 专业陪伴感`

不建议做成：

- 纯工具后台风
- 纯客服聊天风
- 过度花哨的炫技风

建议设计关键词：

- 清晰
- 稳定
- 有秩序
- 有学习氛围
- 能体现“老师”而不是“机器人”

## 3. 品牌视觉建议

### 3.1 色彩

建议采用：

- 主色：深蓝 / 靛蓝
- 辅色：青绿 / 暖橙
- 中性色：浅灰、深灰、白

为什么：

- 教育产品需要可信赖感
- 蓝色系更容易建立专业和稳定心智
- 辅助色用于区分学习状态、老师类型和反馈状态

### 3.2 字体

Web 端建议：

- 中文：思源黑体 / HarmonyOS Sans / 阿里巴巴普惠体
- 英文：Inter 或 Geist

原则：

- 正文可读性优先
- 标题需要有节奏感
- 不用花哨字体破坏专业感

## 4. UI 体系划分

### 4.1 学生端

重点：

- 老师选择清晰
- 聊天内容清晰
- 引用来源清晰
- 学习进度清晰

### 4.2 老师后台

重点：

- 规则编辑效率
- 知识库状态可视化
- 数据反馈直观

### 4.3 管理后台

重点：

- 信息密度高
- 操作反馈明确
- 风险和异常突出显示

## 5. 核心页面设计建议

### 5.1 首页

需要体现：

- 这是 AI 教师市场
- 每个老师有自己的领域和风格
- 产品支持长期记忆和个性化学习

建议结构：

- 顶部价值主张
- 老师卡片矩阵
- 产品能力说明
- 真实示例对话
- 用户评价或效果说明

### 5.2 老师详情页

需要体现：

- 老师定位
- 擅长领域
- 风格标签
- 知识来源
- 适合什么用户

### 5.3 聊天页

聊天页是最重要的页面。

必须体现：

- 当前对话对象是谁
- 当前回答来自哪里
- 用户是否可以切换语言和偏好
- 用户是否能看到记忆状态

建议布局：

- 左侧：历史会话 / 老师信息
- 中间：主聊天区
- 右侧：引用、记忆、学习提示

移动端则改为抽屉和底部切换面板。

### 5.4 记忆管理页

目标：

- 让用户知道系统记住了什么
- 用户能改错
- 用户不会觉得自己被“偷偷记录”

建议展示：

- 学习目标
- 当前水平
- 常见错误
- 偏好设置
- 最近更新来源

### 5.5 知识库页

目标：

- 文件上传清楚
- 索引状态透明
- 权限范围清楚

建议展示：

- 文件列表
- 上传时间
- 处理状态
- 所属范围：公开 / 私有
- 语言和主题标签

## 6. 设计系统建议

建立基础设计系统，至少包含：

- 色板
- 字体系统
- 间距系统
- 圆角系统
- 阴影系统
- 按钮规范
- 表单规范
- 卡片规范
- 聊天气泡规范
- 状态颜色规范

## 7. 组件设计建议

### 7.1 基础组件

- Button
- Input
- Select
- Modal
- Drawer
- Tabs
- Badge
- Tooltip

### 7.2 业务组件

- TeacherCard
- TeacherProfilePanel
- ChatBubble
- CitationCard
- MemoryCard
- RuleConfigPanel
- FileUploadPanel

## 8. 交互设计建议

### 8.1 聊天交互

- 流式输出不能卡顿
- 引用内容可折叠展开
- 文件上传状态要可见
- 错误反馈要明确

### 8.2 后台交互

- 保存和发布分开
- 规则改动要有版本提示
- 高风险操作要二次确认

### 8.3 多端一致性

- 核心颜色和层级一致
- 核心组件语义一致
- 平台差异只保留在交互层，不保留在视觉语言层

## 9. 多语言 UI 规范

- 所有文本长度要预留中英文差异
- 按钮不要依赖中文短文本宽度
- 表单和卡片要适应语言切换后的换行
- 聊天气泡布局要支持中英混排

## 10. UI 开发协作方式

建议交付物：

- Figma 页面稿
- 组件库规范
- 交互标注
- 颜色与字体系
- 响应式规则

前后端协作时要提前定义：

- 引用块数据结构
- 记忆项展示结构
- 老师卡片字段
- 知识库状态字段

## 11. UI 人员配置建议

### 最低配置

- 1 名 UI/UX 设计师

### 更合理配置

- 1 名 产品设计师，负责学生端和品牌
- 1 名 中后台设计师，负责老师后台和平台后台

## 12. UI 风险点

- 做得太像普通聊天机器人，差异感不足
- 做得太像后台系统，缺少教育陪伴感
- 记忆和引用不展示清楚，用户难以建立信任
- 多端视觉不统一，品牌感会被打散

## 13. 最现实的 UI 建议

- 先把学生端聊天页和老师市场页设计透
- 再做老师后台
- 最后做平台后台
- 不要一开始试图把所有页面都画满

真正决定产品气质的，不是后台页，而是：

- 首页
- 老师详情页
- 聊天页
- 记忆页
